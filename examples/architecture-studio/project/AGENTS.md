# Architecture Studio Project Instructions

This project is a fictional studio example. Keep every entry generic, role-based, and safe to share.

## Scope

Track three related projects:

- Project Atrium: office reception, acoustic treatment, and surface palette.
- Project Garden Flat: kitchen and living-room refresh with durable finishes.
- Project Lantern Row: entry, corridor lighting, and storage improvement.

Use collaborator roles instead of people names:

- Lead Designer
- Materials Lead
- Lighting Consultant
- Site Coordinator
- Client Sponsor

## Memory Rules

- Put local vocabulary in `memory/glossary.md`.
- Put design decisions in `memory/decisions.md`.
- Put reusable lessons in `memory/wiki-queue.md` before promoting them to the global wiki.
- Promote only lessons that apply across at least two projects.
- Keep global wiki entries client-neutral and reusable by another studio project.

## Sensitive Detail Handling

Never store:

- Exact addresses or floor plans.
- Client, neighbor, contractor, or supplier names.
- Access codes, alarm details, keys, or site schedules.
- Exact budgets, supplier quotes, invoices, or private photos.
- Personal routines beyond a generalized design need.

Use these replacements:

| Sensitive detail | Safe replacement |
| --- | --- |
| Exact address | "urban corner unit" or "townhouse entry" |
| Client name | "Client Sponsor" or "client household" |
| Supplier quote | "mid-range material quote" |
| Access code | `[BLOCKED: site access detail]` |
| Private routine | "morning traffic" or "evening use" |

## Style

Keep notes concise. Prefer decision records and reusable lessons over long narrative. If a detail feels private but may affect design, generalize the design constraint and block the original detail.
